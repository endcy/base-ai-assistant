# RPC 接口契约模块

> 定义微服务间的通信契约，包含接口协议、请求/响应 DTO、枚举常量等。

## 模块定位

本模块是微服务间通信的"合同层"，定义了核心 AI 服务与管理后台、外部调用方之间的接口协议和数据传输格式。所有跨模块调用的接口签名、请求参数、返回结构均在此模块统一声明，确保各模块间的契约一致性。

## 核心能力

### 接口契约定义

- **AI 问答接口**：定义同步问答、流式问答、RAG 文档匹配、简单对话等接口协议
- **文档管理接口**：定义文档增删改查、状态变更、批量操作等接口协议
- **对话记录接口**：定义按对话 ID 查询历史记录等接口协议
- **测试接口**：定义开发测试用的模拟问答接口

### 请求数据模型

- **知识问答查询参数**：封装对话 ID、用户问题、查询类型、知识领域、业务类型等请求参数
- **文档操作参数**：封装文档创建/更新/删除/启停等操作所需的参数
- **文档匹配参数**：封装 RAG 文档匹配查询的参数
- **简单对话参数**：封装简单对话场景的请求参数

### 响应数据模型

- **AI 回答结果**：包含回答文本、相关文档列表、Token 用量统计
- **流式响应块**：SSE 流式推送的文本块、文档块、Token 块
- **文档匹配结果**：包含匹配文档列表及相似度评分
- **文档状态结果**：包含文档状态列表及操作结果
- **对话记录结果**：包含历史问答记录详情

### 通用基础模型

- **统一请求基类**：所有请求 DTO 的基础结构（时间戳等公共字段）
- **统一响应基类**：所有响应 DTO 的基础结构（状态码、消息、数据体）
- **通用请求/响应包装器**：带版本号和泛型参数的请求/响应封装
- **流式响应模型**：SSE 流式推送的统一数据结构

### 枚举常量

- **接口响应状态**：成功、失败、错误等标准状态码
- **问答类型**：普通 RAG 问答、DeepSeek 搜索等查询类型
- **文档操作类型**：启用、禁用、刷新、删除等操作类型枚举
- **消息类型**：流式响应中的文本、文档、Token 等消息类型

### 常量定义

- **RPC 配置常量**：服务调用相关的配置键和默认值

## 设计原则

- **接口先行**：先定义契约再实现，确保各模块开发对齐
- **DTO 与 Entity 分离**：接口层使用独立 DTO，不直接暴露数据库实体
- **统一响应格式**：所有接口返回统一包装结构，便于调用方统一处理
- **流式支持**：同时支持同步请求/响应和 SSE 流式推送两种模式

## 依赖

- 无内部模块依赖（保持轻量，便于外部工程引用）
- Fastjson（JSON 序列化）
- Lombok（代码简化）
- Spring Web / WebFlux（HTTP 通信基础）
- Hutool（工具类）
