# MCP 服务模块

> Model Context Protocol 工具定义与发现模块，负责本地工具注册和远程 MCP 服务连接管理。

## 模块定位

本模块是 AI 与外部系统通信的"桥梁"，实现了 MCP（Model Context Protocol）协议的客户端与服务端能力。通过本模块，大模型可以发现并调用本地定义的工具函数，也可以连接远程 MCP 服务获取外部实时数据。

## 核心能力

### MCP 服务端

- **本地工具注册**：将本模块中定义的工具函数暴露为 MCP Server 端点，供外部应用通过 MCP 协议调用
- **Streamable HTTP 协议**：支持最新的 Streamable HTTP 传输模式，具备断线重连能力，推荐生产环境使用
- **SSE 兼容**：同时支持传统的 Server-Sent Events 传输模式

### MCP 客户端

- **远程服务发现**：自动扫描并连接配置中的远程 MCP 服务器
- **多连接模式**：同时支持 SSE 和 Streamable HTTP 两种远程连接方式
- **连接属性配置**：通过配置类管理远程 MCP 服务的 URL、超时等参数
- **工具自动注入**：远程 MCP 服务提供的工具自动注册为 Spring AI 的 ToolCallback

### 内置 MCP 工具示例

- **图片搜索**：通过 Pexels API 搜索并返回图片
- **电价查询**：查询电力市场实时/历史电价信息
- **订单查询**：查询充放电业务订单数据
- **站点设备查询**：查询场站和设备信息
- **用户信息查询**：查询用户基本信息和余额

## 工具开发说明

新增 MCP 工具只需：

1. 在工具类中使用 `@ToolMapping` 注解声明工具名称、标题和描述
2. 工具描述需包含调用场景、触发关键词等详细信息
3. 模块启动时自动扫描并注册到 MCP Server

## 配置说明

- MCP 服务端开关（启用/禁用）
- 服务端协议类型（streamable-http / sse）
- 远程 MCP 连接列表（URL、类型、超时时间）
- 客户端开关（启用/禁用）
