# 数据持久化模块

> 统一管理所有数据库操作，支持 MySQL 业务库 + PostgreSQL 向量库双数据源架构。

## 模块定位

本模块是整个框架的数据层，封装了所有与数据库交互的逻辑。采用双数据源设计：MySQL 负责业务数据（知识文档、对话记录、分类配置等）的持久化，PostgreSQL + PGVector 负责文档向量数据的存储与检索。对上层模块提供标准化的
Service 接口，屏蔽底层数据库细节。

## 核心能力

### 知识文档管理

- **文档全生命周期 CRUD**：创建、查询、更新、删除知识文档
- **文档状态流转**：管理文档的上架/下架/向量化状态
- **分页加载未向量化文档**：支持大批量文档分页加载，防止内存溢出
- **按分类查询**：支持按知识领域和业务类型筛选文档
- **文档批量操作**：支持批量启停、批量删除并级联清理向量数据

### 向量存储服务

- **向量相似度检索**：基于 PGVector 的余弦距离检索
- **BM25 全文检索评分**：结合 pg_jieba 中文分词的 BM25 评分计算
- **向量数据清理**：按文档 ID 精准删除对应的向量数据
- **向量存在性检测**：判断指定文档是否已完成向量化
- **元数据过滤**：支持按知识领域、业务类型、分组等多维度过滤

### 对话记录管理

- **对话持久化**：用户问答记录的存储与查询
- **相关度反馈**：支持对问答结果评分
- **按对话 ID 检索**：快速定位同一轮对话的所有记录

### 知识分类配置

- **分类配置 CRUD**：两级分类（领域 + 业务）的增删改查
- **分类缓存**：基于 Spring Cache 的分类数据缓存，减少数据库压力
- **按类型查询**：按分类类型（scope/business）筛选

### 数据模型

#### MySQL 业务表

| 表名        | 用途                     |
|-----------|------------------------|
| 知识文档表     | 存储文档标题、内容、分类、状态、版本等元信息 |
| 对话记录表     | 存储用户问答对、评分、Token 用量等   |
| 分类配置表     | 存储知识领域和业务领域的分类定义       |
| Token 用量表 | 存储每次请求的 Token 消耗明细     |

#### PostgreSQL 向量表

| 表名    | 用途                                        |
|-------|-------------------------------------------|
| 向量存储表 | 存储文档分片后的向量数据（1024 维），支持 HNSW 索引和 GIN 全文索引 |

### 数据转换

- **MapStruct 转换器**：Entity ↔ DTO 自动映射，避免手动 set 转换
- **查询参数封装**：统一的查询条件封装，支持分页、排序、条件过滤
- **向量数据模型**：封装向量文档的 ID、内容、元数据和相似度分数

## 技术特性

- **双数据源隔离**：MySQL 和 PostgreSQL 独立事务管理，互不干扰
- **MyBatis-Plus 增强**：继承基础 Service/Mapper 能力，支持通用 CRUD
- **PGVector 类型处理**：自定义 MyBatis TypeHandler，自动处理向量类型的读写
- **查询上下文**：封装 RAG 查询时的知识领域、业务类型、原始问题、改写问题等信息
