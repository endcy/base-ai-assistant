# 领域模型模块

> 全局共享的领域定义层，统一管理枚举、业务对象、请求参数等领域模型。

## 模块定位

本模块是整个框架的"语言层"，定义了各业务模块共同使用的领域概念和数据传输结构。通过将枚举、业务对象等公共模型集中管理，确保各模块对业务概念的理解一致，避免重复定义和循环依赖。

## 核心能力

### 领域枚举

- **文档来源类型**：标识知识文档的存储来源（本地文件、数据库、云知识库、向量库）
- **数据来源预测**：意图分析 Agent 输出的数据源预测结果（本地文档、向量检索、云知识库、业务数据表、未知）
- **知识领域类型**：知识文档的领域分类（如充电运营、能源管理、设备运维等示例分类）
- **业务领域类型**：知识文档的业务分类（如用户客服、商户客服、运营管理等示例分类）
- **内容类型**：文档内容的格式类型（Markdown、纯文本等）
- **消息类型**：MQ 消息队列的消息类型标识

### 业务对象

- **基础 MQ 消息**：消息队列消息的公共基础结构（消息 ID、时间戳、消息类型等）
- **扩展 MQ 消息**：携带具体业务数据的消息体封装

### 请求参数

- **问题格式化参数**：用户提问时的格式化配置（是否携带上下文、是否启用意图分析等）

### 常量定义

- **对象转换常量**：对象到文本转换过程中的公共配置（默认 Agent 名称、知识标题标识等）

## 设计原则

- **零业务逻辑**：本模块仅包含数据定义，不包含任何业务处理逻辑
- **最小依赖**：仅依赖 Lombok、MapStruct、Fastjson 等轻量库，不引入 Spring 框架
- **跨模块共享**：被所有业务模块依赖，是工程依赖树的最底层模块之一
- **枚举可扩展**：关键枚举提供 UNKNOWN 兜底值，分类信息优先从数据库配置表动态加载

## 使用场景

| 使用方      | 使用内容               |
|----------|--------------------|
| 核心 AI 服务 | 数据来源预测枚举、消息对象、请求参数 |
| 管理后台     | 领域枚举、业务对象          |
| 持久化层     | 领域枚举（用于元数据过滤）      |
| RPC 契约   | 消息类型枚举、请求/响应基础结构   |
| 通用服务     | 枚举引用               |
